Febrl-0.4 installation instructions
===================================

Peter Christen, 2 November 2007


The basic Febrl-0.4 installation requires Python 2.5, with additional
modules and libraries needed for the graphical user interface (GUI)
and the support vector machine (SVM) classification method.

Python 2.5:
  http://www.python.org

For the GUI, PyGTK and Matplotlib are required:
  http://www.pygtk.org
  http://matplotlib.sourceforge.net

The SVM classifier is based on libsvm:
  http://www.csie.ntu.edu.tw/~cjlin/libsvm/

  Follow the installation instruction within the python directory once
  you have downloaded and unpacked libsvm.


Windows: There is a one-in-all installer for Python and PyGTK available
  at: http://osl.ulpgc.es/~arc/gnome/pygtk-setup.exe
  For more details please see: http://www.pygtk.org/

Macintosh: For information on running PyGTK on Max OSX please see:
http://faq.pygtk.org/index.py?req=show&file=faq01.019.htp


Test of installed modules:
--------------------------

To test if the required libraries are installed in your Python
distribution, start Python and try the following:

>>> import matplotlib
>>> matplotlib.use('GTK')

>>> import pygtk
>>> pygtk.require("2.0")

>>> import gtk
>>> import gtk.glade

>>> import svm

None of these import commands should give you an error.


Febrl-0.4 installation:
-----------------------

Unpack the febrl-0.4.tar.gz or febrl-0.4.zip archive and a new
directory named 'febrl-0.4' will be created containing all the
necessary Febrl modules and additional files such as example data sets,
documentation, the Febrl data set generator and testing programs.

Go into the 'tests' sub-directory within 'febrl-0.4' and either run
all tests using the corresponding script provided:

#> ./allTests.sh

or run the tests individually, for examples:

#> python indexingTest.py
#> python datasetTest.py


Starting Febrl-0.4
------------------

The Febrl GUI can be started using:

#> ./guiFebrl.py

(you might have to set the execute permission for the file guiFebrl.py
to allow execute my the user), or using:

#> python guiFebrl.py

Traditional Febrl project modules (either implemented manually or
saved from within the GUI) can be run outside the GUI, for example:

#> python my-linkage-project.py


Problems and errors:
--------------------

Please note that this is the initial 'alpha' distribution of Febrl-0.4
which has only been tested to a limited extend on an Ubuntu Linux
platform.

Please report and problems and bugs to: peter.christen@anu.edu.au
